================================================================================
                            IMPLEMENTED WORKFLOW NOTES
================================================================================

ARTICLE
  - Journalists create articles in draft status.
  - Journalists can edit only their own draft or rejected articles.
  - Journalists submit their own articles for editorial review.
  - Editors can preview, edit, approve, reject, or delete articles.
  - Published articles are visible to the public.

COMMENT
  - Only readers can post comments.
  - Comments are allowed only on published articles.
  - No public edit/delete UI is implemented for comments.

PUBLISHER
  - Publishers are organizations, not a separate application role.
  - Only editors create publishers through the application UI.

SUBSCRIPTION
  - Only readers can subscribe or unsubscribe.
  - A subscription targets either one publisher or one journalist.
  - Readers can view their own subscriptions.

NOTIFICATION
  - Notifications are created automatically when a subscribed article is published.
  - Readers view notifications from the reader dashboard.

NEWSLETTER
  - A newsletter model exists in the admin.
  - No dedicated public newsletter workflow is implemented in the current UI.
